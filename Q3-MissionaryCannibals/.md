# Search Algorithm’s Comparison

## Breadth First Search (BFS)

Breadth First Search guarantees the shortest solution when all edge costs are equal.  
It explores nodes level by level starting from the root node.

**Characteristics**

- Uses a **queue** data structure
- **Complete** – always finds a solution if one exists
- **Optimal** when all step costs are equal
- Requires **large memory** because many nodes are stored

---

## Depth First Search (DFS)

Depth First Search explores nodes as deeply as possible before backtracking.

**Characteristics**

- Uses a **stack** data structure
- Uses **less memory than BFS**
- **Not optimal** because it may find a longer path
- **Not complete** in infinite depth spaces
- May get stuck exploring deep paths

---

## Depth-Limited Search (DLS)

Depth-Limited Search is a modified version of DFS where a **maximum depth limit** is applied.

**Characteristics**

- Prevents infinite depth exploration
- Uses a **depth limit to control search**
- **Not optimal**
- Solution may not be found if it lies **beyond the depth limit**

---

## Iterative Deepening Depth First Search (IDDFS)

Iterative Deepening DFS repeatedly performs DFS with increasing depth limits.

**Characteristics**

- Combines advantages of **BFS and DFS**
- **Complete**
- **Optimal** for equal step costs
- Uses **less memory than BFS**
- Repeats searches but remains efficient

---

## Uniform Cost Search (UCS)

Uniform Cost Search expands nodes based on the **lowest path cost** rather than depth.

**Characteristics**

- Uses a **priority queue**
- Expands the **lowest-cost node first**
- **Complete**
- **Optimal**
- Can be **slower** because it explores many nodes
